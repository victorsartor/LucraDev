# 💰 LucraDev | CommitRate Precision

> **Engenharia financeira para desenvolvedores: transforme seu código em lucro real.**

O **LucraDev** é uma ferramenta estratégica de precificação para freelancers. O projeto nasceu da necessidade de converter custos operacionais complexos (hardware, impostos, ociosidade e metas de investimento) em um valor de hora técnica sustentável.

---

## 🚀 O Diferencial Técnico

Diferente de calculadoras genéricas, o LucraDev utiliza uma arquitetura baseada em **Separação de Preocupações (SoC)**:

* **Engine de Cálculo:** Uma lógica isolada que processa diluição anual (férias/13º) e depreciação de ativos.
* **Persistência Reativa:** Integração com `localStorage` para manter os dados do usuário seguros entre sessões.
* **UX Stepper:** Interface dividida em fluxos lógicos para reduzir a carga cognitiva no preenchimento de dados financeiros.

## 🛠️ Stack Tecnológica

* **HTML5 Semântico:** Foco em acessibilidade.
* **CSS3 Moderno:** Uso de CSS Variables para fácil manutenção e Grid/Flexbox para total responsividade.
* **Vanilla JavaScript (ES6+):** Manipulação de DOM e lógica de estado sem dependências externas.
* **CGI Integration:** Hero Section otimizada com renderizações de alta fidelidade para uma experiência visual premium.

## 📊 Regras de Negócio Implementadas

1.  **Buffer de Ociosidade:** Adição automática de 20% sobre as horas para cobrir prospecção e reuniões.
2.  **Diluição de Equipamento:** Cálculo de amortização de hardware em ciclos de 36 meses.
3.  **Reserva de Emergência:** Campo dedicado para cálculo de ROI (Retorno sobre Investimento) pessoal.

## 🛠️ Metodologia e Desenvolvimento

Este projeto foi concebido como um desafio técnico pessoal para consolidar e validar conhecimentos avançados em **HTML5 Semântico** e **CSS3 Moderno (Flexbox/Grid)**, focando em uma interface funcional e totalmente responsiva.

A arquitetura do sistema foi planejada para ir além do básico, aplicando conceitos de:
* **Lógica de Negócio Real:** Implementação de cálculos de amortização e margem de ociosidade.
* **Clean Code & Refatoração:** Durante o desenvolvimento, utilizei o **Gemini** como um parceiro de revisão de código para auditar a estrutura das funções, garantindo que o JavaScript seguisse padrões de código limpo, legibilidade e manutenibilidade.
* **Arquitetura Modular:** Separação estrita entre lógica matemática (Business Logic) e manipulação de interface (UI Logic), facilitando futuras expansões do sistema.
